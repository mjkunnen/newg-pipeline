    **Step 1 — Add checkTokenType() to fromPostgres.ts (per D-11):**

    Add an exported async function `checkTokenType(token: string)` in fromPostgres.ts:
    ```typescript
    export async function checkTokenType(token: string): Promise<void> {
      const GRAPH_API = "https://graph.facebook.com/v23.0";
      try {
        const resp = await fetch(`${GRAPH_API}/me?fields=name,id&access_token=${token}`);
        if (!resp.ok) {
          console.warn("[launcher] Could not verify token type — Meta /me returned", resp.status);
          return;
        }
        const data: { name?: string; id?: string } = await resp.json();
        const name = data.name || "";
        if (/system|bot|api|automation/i.test(name)) {
          console.log(`[launcher] Token type: System User confirmed (name="${name}")`);
        } else {
          console.warn(`[launcher] WARNING: Token may be a personal token (name="${name}"). System User token recommended — see docs/META_TOKEN_SETUP.md`);
        }
      } catch (err) {
        console.warn("[launcher] Token type check failed:", err instanceof Error ? err.message : String(err));
      }
    }
    ```

    Call `checkTokenType(process.env.META_ACCESS_TOKEN!)` early in `main()` (after env var validation, before fetching items). Make it non-blocking — await it but catch errors (it should never prevent the launch from running). Per D-11, this is informational only, not a gate.

    **# Meta System User Token Setup**

    **## Why**
    Personal access tokens expire every 60 days, silently breaking the automated launch workflow. System User tokens never expire.

    **## Prerequisites**
    - Admin access to Meta Business Manager (business.facebook.com)
    - The ad account (META_AD_ACCOUNT_ID) must be in this Business Manager

    **## Steps**
    1. Go to Business Manager -> Settings -> Users -> System Users
    2. Click "Add" to create a new System User (name: "NEWG Pipeline", role: Admin)
    3. Click "Generate New Token" for the System User
    4. Select the app associated with your ad account
    5. Grant these permissions: `ads_management`, `ads_read`, `pages_read_engagement`, `pages_manage_ads`, `business_management`
    6. Copy the generated token

    **## Where to Set the Token**
    - Railway: ad-command-center service -> Variables -> META_ACCESS_TOKEN
    - GitHub Actions: Repository -> Settings -> Secrets -> META_ACCESS_TOKEN
    - Local development: `.env` file -> META_ACCESS_TOKEN=<token>

    **## Verification**
    Run: `curl "https://graph.facebook.com/v23.0/me?fields=name,id&access_token=YOUR_TOKEN"`
    Expected: `{"name": "NEWG Pipeline", "id": "..."}`
    If the name is a human name, you're still using a personal token.

    **## Token Rotation**
    System User tokens don't expire, but you can regenerate them at any time from Business Manager -> System Users -> Generate New Token. Update all three locations (Railway, GitHub, .env) when you rotate.
