# NEWG Creative Hub — Design Spec

## Problem

Competitor ads scraped from PPSpy need to be remade by a creative editor (Jerson) and launched on Meta/TikTok. Currently there's no interface for assigning work, tracking progress, or collecting deliverables. PPSpy compresses videos, so Jerson needs links to download originals in full quality.

## Solution

Static HTML dashboard hosted on GitHub Pages. Generated daily by the scraper pipeline. Jerson opens the URL, sees his daily assignments, downloads originals from PPSpy, makes remakes, and submits deliverables via a form. Submissions go to a Google Sheet via Zapier webhook.

## Users

- **Jerson** — creative editor, makes remakes daily
- **Max** — owner, reviews submissions, triggers batch launches

## Screens

### 1. Login

- Name selection: Jerson / Max (stored in localStorage)
- Minimal screen: NEWG logo, "Creative Hub" subtitle, name buttons
- No authentication — just name selection for personalization

### 2. Datums Overzicht

- Header: NEWG brand + avatar initial
- Title: "Opdrachten"
- List of date cards, newest first
- Per date card:
  - Date (Dutch format: "Zondag 23 maart")
  - "VANDAAG" tag on today's date
  - Count: "2/5" with large numbers
  - Meta: "5 creatives · 3 video, 2 image"
  - Progress bar (orange = partial, green = complete)
- Click a date → navigates to day detail

### 3. Dag Detail

- Header: back button + date
- Progress bar + "2 / 5 klaar" counter
- List of ad cards, sorted by reach descending

#### Ad Card (open state)

- Thumbnail (72px, from scraped data)
- VIDEO/IMAGE badge
- Ad ID + type label
- Ad copy (2 lines max, truncated)
- Stats: reach (formatted), days active
- **"Bekijk origineel op PPSpy"** link — opens PPSpy search page
- Submission form:
  - Google Drive link (text input)
  - Landing page URL (text input)
  - Platform: Meta / TikTok / Beide (toggle buttons)
  - "Markeer als klaar" button (black)

#### Ad Card (done state)

- Same thumbnail + info but with green "KLAAR ✓" badge
- Collapsed summary: Drive link, landing page, platform
- Green-tinted card border
- Form hidden, summary shown instead

## Data Architecture

### Input: Scraper → Dashboard

Existing `output/raw/{date}/metadata.json` (generated by ppspy.ts scraper):
```
ScrapedAd {
  id, type, creativeUrl, thumbnailUrl, localPath,
  adCopy, reach, daysActive, startedAt, platforms, scrapedAt
}
```

### Dashboard State: `docs/data/{date}.json`

Per-date file generated by dashboard generator:
```json
{
  "date": "2026-03-23",
  "competitor": "decarba",
  "ads": [
    {
      "id": "decarba_2026-03-23_0",
      "type": "video",
      "thumbPath": "thumbs/decarba_2026-03-23_0.jpg",
      "adCopy": "NEW Collection is live...",
      "reach": 303000,
      "reachFormatted": "303K",
      "daysActive": 3,
      "startedAt": "Mar 18 2026",
      "platforms": ["facebook", "instagram"],
      "ppskyUrl": "https://app.ppspy.com/ads?extend_keywords=..."
    }
  ]
}
```

### Output: Submissions → Google Sheet via Zapier

When Jerson clicks "Markeer als klaar", the dashboard sends a POST to a Zapier webhook URL:
```json
{
  "ad_id": "decarba_2026-03-23_0",
  "date": "2026-03-23",
  "drive_link": "https://drive.google.com/file/d/...",
  "landing_page": "https://newgarments.nl/hoodies",
  "platform": "both",
  "submitted_by": "jerson",
  "submitted_at": "2026-03-23T14:30:00Z"
}
```

Zapier writes this to a Google Sheet "NEWG Submissions" with columns:
| Ad ID | Date | Drive Link | Landing Page | Platform | Submitted By | Submitted At | Launched |

### Launch Flow

Daily batch (triggered by Max or scheduled):
1. Read Google Sheet "NEWG Submissions" — filter rows where Launched = empty
2. For each submission: download creative from Drive link
3. Launch on selected platform(s) via Meta Marketing API / TikTok API
4. Mark row as "Launched" in Sheet with timestamp

## Styling

- **Theme**: Light / off-white
- **Background**: `#e8e4df` (page), `#f7f5f2` (cards), `#fff` (inputs/detail cards)
- **Font**: Inter (Google Fonts)
- **Accent**: `#e04400` (orange-red) for branding, links, active states
- **Success**: `#16a34a` (green) for completed items
- **Warning/Open**: `#b8860b` (gold) for open status
- **Buttons**: Black (`#1a1a1a`) for submit, no orange buttons
- **Border radius**: 14-16px for cards, 10-12px for inputs
- **Mobile-responsive**: single column on mobile, max-width 420px phone-frame aesthetic

### Client-side Routing

Single HTML file with hash-based routing:
- `#/` → Login screen
- `#/dates` → Datums overzicht
- `#/day/2026-03-23` → Dag detail

All data loaded from `docs/data/*.json` files via fetch. No build step for the frontend.

### Zapier Webhook

- Dashboard sends POST requests to a Zapier webhook URL
- Webhook URL stored as a variable in the generated HTML (set by generate.ts from env)
- Zapier workflow: Catch Hook → Create Spreadsheet Row in "NEWG Submissions"
- Env var: `ZAPIER_WEBHOOK_URL` in `.env`

### PPSpy Link

Each ad links to PPSpy search with the competitor keyword pre-filled:
```
https://app.ppspy.com/ads?extend_keywords=[{"field":"all","value":"decarba","logic_operator":"and"}]&ad_forecast=[3]&order_by=ad_created_at&direction=desc
```

This is a single link for all ads (PPSpy doesn't have per-ad deep links from the grid view).
